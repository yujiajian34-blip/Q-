## 📥 获取音效文件

### 方式一：从 GitHub 仓库复制（推荐）
从官方仓库获取音效文件：
https://github.com/yujiajian34-blip/q-pet/tree/main/wav

1. 访问上述链接
2. 下载所有 .wav 文件
3. 放入此文件夹中

### 方式二：使用文本转语音工具生成
如果无法获取原始文件，可以用 TTS 工具生成：
- **Windows**: 使用 PowerShell 的 TTS 功能
- **在线工具**: https://tts.baidu.com/ 或其他 TTS 服务
- **Python**: 使用 `pyttsx3` 库

### 方式三：使用空文件（暂时跳过）
如果只是测试项目，可以创建空的 .wav 文件：
```bash
python create_placeholder_wavs.py
```

## 📊 音效文件预期大小

每个音效文件大约 50-200 KB（取决于：
- 音质（建议 128kbps MP3 或 16bit PCM WAV）
- 长度（通常 1-3 秒）

## 🐛 故障排查

如果出现 "音效文件未找到" 错误：

1. **检查文件名**
   - 确保文件名完全匹配（包括中文字符）
   - 检查文件扩展名是否为 `.wav`

2. **检查路径**
   - 确保文件在 `wav/` 文件夹中
   - 不要放在子文件夹里

3. **检查权限**
   - 确保文件有读权限
   - 不要将文件标记为"只读"

4. **文件格式**
   - 确保是有效的 WAV 格式
   - 可以用 ffmpeg 重新编码：
   ```bash
   ffmpeg -i input.wav -acodec pcm_s16le -ar 22050 output.wav
   ```

## 💡 建议

- **音质**: 使用 16-bit PCM WAV 或 128kbps MP3
- **采样率**: 22050 Hz 或 44100 Hz
- **时长**: 每个音效 1-3 秒为佳
